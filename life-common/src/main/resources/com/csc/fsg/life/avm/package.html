The Allowable Values sub-system is the API used for inquiry only access to the AVM database.

Allowable values are identified by a field name in the AVM database. An AVM field can support allowable values as:

  • A list of specified core (internal) and display (english) values.
  • A range of values.

While each application will tie that allowble value field name to an object/field in different ways. Two common approaches are used:

  • The field name is exactly the same as in the object.
  • The field is tied to the object via meta data. The two meta data approaches are:
    • FSConnect Domains
    • Attributes (Apache Commons attributes for Java 1.4.x)

This API supports generic access by AVM field name, and support for acces by class/field name where it will use the com.csc.fsg.life.biz.meta package to access the apache commons attributes.
